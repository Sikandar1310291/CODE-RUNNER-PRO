1 Introduction to Python

What is Python?

Python is a high-level, interpreted, general-purpose programming language. Being high-level means Python abstracts away low-level details like memory management, allowing developers to focus on logic and problem-solving.

Python is also interpreted, meaning code is executed line-by-line, which makes debugging easier and development faster. Its syntax is intentionally designed to be readable and close to plain English, reducing the learning curve significantly.

History of Python

  • Python was created by Guido van Rossum
  • First released in 1991
  • Named after Monty Python (not the snake)

üéØ The Zen of Python

The core philosophy of Python emphasizes:

  • Simplicity
  • Readability
  • Explicitness over cleverness

Python's popularity comes from a combination of simplicity and power:

4 Control Flow Statements

Control flow allows your program to make decisions and repeat actions.

Conditional Statements

python 
age = 20

if age >= 18:
    print("Adult")
elif age > 12:
    print("Teen")
else:
    print("Child")

Note: Python uses indentation instead of braces, enforcing clean and readable code.

match-case (Python 3.10 Feature)

Python 3.10 introduced structural pattern matching:

python 
day = "Monday"

match day:
    case "Monday":
        print("Start of the week")
    case "Friday":
        print("Weekend is near!")
    case _:
        print("Regular day")

This is similar to switch statements in other languages but more powerful.

Loops in Python

for Loop

python 
# Basic for loop
for i in range(5):
    print(i)  # Prints 0, 1, 2, 3, 4

# Loop through a list
fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print(fruit)

Used when the number of iterations is known.

while Loop

python 
count = 0

while count < 5:
    print(f"Count is: {count}")
    count += 1

Used when looping depends on a condition.

5 Python Data Structures

Data structures help organize and store data efficiently.

Lists

Lists are mutable, ordered, and widely used.

python 
# Creating a list
numbers = [1, 2, 3, 4]

# Adding elements
numbers.append(5)

# Accessing elements
first = numbers[0]  # Gets 1

# List methods
numbers.remove(2)  # Removes value 2
numbers.pop()      # Removes last element

Tuples

Tuples are immutable, making them safer for fixed data.

python 
# Creating a tuple
point = (10, 20)

# Accessing elements
x = point[0]  # Gets 10
y = point[1]  # Gets 20

# Tuples are immutable (cannot be changed)
# point[0] = 15  # This would cause an error

Sets

Sets store unique values and are unordered.

python 
# Creating a set
unique = {1, 2, 3, 3, 3}  # Duplicates are removed
print(unique)  # Output: {1, 2, 3}

# Set operations
set1 = {1, 2, 3}
set2 = {3, 4, 5}
union = set1 | set2          # {1, 2, 3, 4, 5}
intersection = set1 & set2    # {3}

Dictionaries

Dictionaries store data in key-value pairs and are extremely fast.

python 
# Creating a dictionary
user = {
    "name": "Ali",
    "age": 20,
    "email": "ali@example.com"
}

# Accessing values
name = user["name"]  # Gets "Ali"
age = user.get("age")  # Safer way to get value

# Adding/updating values
user["city"] = "New York"
user["age"] = 21

# Looping through dictionary
for key, value in user.items():
    print(f"{key}: {value}")

List Comprehension

A concise and efficient way to create lists:

python 
# Traditional way
squares = []
for x in range(5):
    squares.append(x * x)

# List comprehension (better)
squares = [x * x for x in range(5)]
# Result: [0, 1, 4, 9, 16]

# With condition
even_squares = [x * x for x in range(10) if x % 2 == 0]
# Result: [0, 4, 16, 36, 64]

6 Functions & Modules

Defining Functions

python 
# Basic function
def greet(name):
    return f"Hello {name}"

# Calling the function
message = greet("Python")
print(message)  # Output: Hello Python

Functions help organize code, improve readability, and enable reuse.

Function Arguments

Python supports multiple types of arguments:

python 
# Positional arguments
def add(a, b):
    return a + b

# Default arguments
def greet(name="Guest"):
    return f"Hello {name}"

# Keyword arguments
def create_user(name, age, city="Unknown"):
    return {"name": name, "age": age, "city": city}

user = create_user(name="Ali", age=25, city="NYC")

# Variable arguments (*args)
def total(*numbers):
    return sum(numbers)

result = total(1, 2, 3, 4, 5)  # Returns 15

# Keyword variable arguments (**kwargs)
def print_info(**info):
    for key, value in info.items():
        print(f"{key}: {value}")

print_info(name="Ali", age=20, city="NYC")

Modules & Imports

Modules allow code reuse across files:

python 
# Import entire module
import math
result = math.sqrt(16)

# Import specific function
from datetime import date
today = date.today()

# Import with alias
import numpy as np
arr = np.array([1, 2, 3])

# Import all (not recommended)
from math import *

Python's standard library provides thousands of ready-to-use modules.

7 Object-Oriented Programming (OOP)

OOP allows you to model real-world entities using code.

Creating Classes

python 
# Defining a class
class Person:
    def __init__(self, name, age):
        self.name = name  # Instance variable
        self.age = age
    
    def introduce(self):
        return f"Hi, I'm {self.name} and I'm {self.age} years old"

# Creating objects (instances)
person1 = Person("Ali", 25)
person2 = Person("Sara", 30)

print(person1.introduce())
# Output: Hi, I'm Ali and I'm 25 years old

A class is a blueprint; objects are instances of that blueprint.

Inheritance

Inheritance promotes code reuse and logical structure.

python 
# Parent class
class Animal:
    def __init__(self, name):
        self.name = name
    
    def speak(self):
        pass

# Child class
class Dog(Animal):
    def speak(self):
        return f"{self.name} says Woof!"

class Cat(Animal):
    def speak(self):
        return f"{self.name} says Meow!"

# Using inheritance
dog = Dog("Buddy")
cat = Cat("Whiskers")

print(dog.speak())  # Output: Buddy says Woof!
print(cat.speak())  # Output: Whiskers says Meow!

Core OOP Concepts

  • Encapsulation ‚Äì Protecting data using private/public access
  • Inheritance ‚Äì Reusing code from parent classes
  • Polymorphism ‚Äì One interface, many forms
  • Abstraction ‚Äì Hiding implementation details

Magic Methods

Magic methods define how objects behave internally:

python 
class Book:
    def __init__(self, title, author):
        self.title = title
        self.author = author
    
    def __str__(self):
        return f"{self.title} by {self.author}"
    
    def __repr__(self):
        return f"Book('{self.title}', '{self.author}')"
    
    def __len__(self):
        return len(self.title)

# Using magic methods
book = Book("1984", "George Orwell")
print(book)         # Calls __str__
print(len(book))    # Calls __len__

8 Exception Handling

Exception handling prevents program crashes and improves reliability.

Basic Exception Handling

python 
try:
    x = int(input("Enter a number: "))
    result = 100 / x
    print(f"Result: {result}")
except ValueError:
    print("Invalid input! Please enter a valid number.")
except ZeroDivisionError:
    print("Cannot divide by zero!")
except Exception as e:
    print(f"An error occurred: {e}")
finally:
    print("Execution completed.")

Raising Exceptions

python 
def validate_age(age):
    if age < 0:
        raise ValueError("Age cannot be negative")
    if age > 150:
        raise ValueError("Age seems unrealistic")
    return age

try:
    validate_age(-5)
except ValueError as e:
    print(f"Error: {e}")

Custom Exceptions

python 
class InsufficientFundsError(Exception):
    pass

class BankAccount:
    def __init__(self, balance):
        self.balance = balance
    
    def withdraw(self, amount):
        if amount > self.balance:
            raise InsufficientFundsError("Not enough funds")
        self.balance -= amount
        return self.balance

# Using custom exception
account = BankAccount(100)
try:
    account.withdraw(150)
except InsufficientFundsError as e:
    print(f"Error: {e}")

Custom exceptions allow precise error handling in large applications.

9 File Handling in Python

File handling enables programs to store, retrieve, and process data permanently.

Reading Files

python 
# Reading entire file
with open("file.txt", "r") as f:
    data = f.read()
    print(data)

# Reading line by line
with open("file.txt", "r") as f:
    for line in f:
        print(line.strip())

# Reading all lines into a list
with open("file.txt", "r") as f:
    lines = f.readlines()
    print(lines)

Why use 'with' statement?

  • ‚úÖ Automatically closes the file
  • ‚úÖ Prevents memory leaks
  • ‚úÖ Prevents file corruption
  • ‚úÖ Best practice in professional code

Writing Files

python 
# Writing to a file (overwrites existing content)
with open("file.txt", "w") as f:
    f.write("Hello, World!\n")
    f.write("Welcome to Python")

# Writing multiple lines
lines = ["Line 1\n", "Line 2\n", "Line 3\n"]
with open("output.txt", "w") as f:
    f.writelines(lines)

Note: "w" mode overwrites the file. If the file doesn't exist, it creates it.

Appending to Files

python 
# Appending to a file
with open("file.txt", "a") as f:
    f.write("\nThis is a new line")

# Data is added without deleting existing content

Working with JSON Files

JSON is widely used for APIs and configuration files.

python 
import json

# Writing JSON
data = {
    "name": "Ali",
    "age": 20,
    "skills": ["Python", "JavaScript", "SQL"]
}

with open("data.json", "w") as file:
    json.dump(data, file, indent=4)

# Reading JSON
with open("data.json", "r") as file:
    loaded_data = json.load(file)
    print(loaded_data["name"])  # Output: Ali

# Converting to/from JSON strings
json_string = json.dumps(data)
python_dict = json.loads(json_string)

JSON allows structured, readable data storage perfect for configuration and API communication.

File Modes Summary

Mode Description Creates if Missing
'r' Read (default) No (error if missing)
'w' Write (overwrites) Yes
'a' Append Yes
'r+' Read and Write No
'rb' Read Binary No
'wb' Write Binary Yes
