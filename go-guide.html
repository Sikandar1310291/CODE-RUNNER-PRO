1 Introduction: Why Go?

Go (often called Golang) was designed at Google by Robert Griesemer, Rob Pike, and Ken Thompson in 2007. They wanted a language that had the efficiency of C++ but the simplicity of Python.

‚ö° Fast Compilation

Go compiles to machine code in seconds, making the development cycle incredibly fast.

üßµ Native Concurrency

Built-in "Goroutines" allow millions of concurrent tasks with minimal memory usage.

üß© Simple Type System

No complex class hierarchies. Go uses composition and interfaces for flexibility.

To start coding in Go, you need to install the Go toolchain from go.dev.

3 Basic Syntax & Variables

Go is statically typed but feels dynamic thanks to type inference.

Variable Declaration

go 
package main
import "fmt"

func main() {
    // 1. Explicit declaration
    var name string = "CodeRunner"
    
    // 2. Short-hand (Type inference) - inside functions only
    version := 1.2
    
    // 3. Constant
    const Platform = "Web"

    fmt.Printf("%s v%.1f on %s\n", name, version, Platform)
}

4 Arrays, Slices & Maps

Understanding the difference between Arrays (fixed size) and Slices (dynamic) is key to Go.

go 
// Slices are dynamic
fruits := []string{"Apple", "Banana"}
fruits = append(fruits, "Orange")

// Maps are Key-Value stores
userAges := map[string]int{
    "Alice": 25,
    "Bob":   30,
}

// Accessing a map
age, exists := userAges["Alice"]

5 Pointers & Memory

Go has pointers, but no pointer arithmetic (so it's safer than C).

go 
func updateName(n *string) {
    *n = "Antigravity" // Dereference and update
}

func main() {
    name := "Original"
    updateName(&name) // Pass memory address
    fmt.Println(name) // Prints: Antigravity
}

6 Structs & Methods

Structs are collections of fields. Methods look like functions but have a "receiver".

go 
type Rectangle struct {
    Width, Height float64
}

// Method with a receiver (r Rectangle)
func (r Rectangle) Area() float64 {
    return r.Width * r.Height
}

func main() {
    rect := Rectangle{10, 5}
    fmt.Println(rect.Area())
}

7 Interfaces & Composition

Interfaces define behavior. In Go, they are implemented implicitly. If a struct has the methods, it implements the interface.

go 
type Shape interface {
    Area() float64
}

func printArea(s Shape) {
    fmt.Println("Area:", s.Area())
}

// Rectangle from before implicitly implements Shape

8 Concurrency: Goroutines & Channels

Concurrency is the heart of Go. Don't communicate by sharing memory; share memory by communicating.

üöÄ Goroutines

A Goroutine is a lightweight thread managed by the Go runtime. Just add the go keyword before a function call.

go 
func fetchURL(url string, c chan string) {
    // Simulate fetching
    c <- "Fetched: " + url
}

func main() {
    msgChan := make(chan string)
    
    go fetchURL("google.com", msgChan)
    go fetchURL("github.com", msgChan)
    
    fmt.Println(<-msgChan) // Block until first message
    fmt.Println(<-msgChan) // Block until second message
}

Select Statement

The select statement lets a goroutine wait on multiple communication operations.

9 Better Error Handling

Go doesn't use try/catch blocks. Instead, it returns errors as values.

go 
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, fmt.Errorf("division by zero")
    }
    return a / b, nil
}

func main() {
    result, err := divide(10, 0)
    if err != nil {
        fmt.Println("Error:", err)
        return
    }
    fmt.Println(result)
}
