1 Introduction: What is SQL?

SQL (Structured Query Language) is the standard language for dealing with Relational Databases (RDBMS). It allows you to store, retrieve, manage, and manipulate data across systems like MySQL, PostgreSQL, Oracle, and SQL Server.

üîç Declarative

Tell the database what you want, not how to get it. SQL engines handle the logic.

üõ°Ô∏è Data Integrity

Ensure your data is accurate and consistent through foreign keys and constraints.

üöÄ Scalable

Modern SQL databases can handle billions of rows with proper indexing and schema design.

2 DDL: Creating & Altering Tables

Data Definition Language (DDL) is used to define the database schema.

sql 
-- 1. Create a table
CREATE TABLE Users (
    UserID INT PRIMARY KEY AUTO_INCREMENT,
    Username VARCHAR(50) NOT NULL UNIQUE,
    Email VARCHAR(100),
    CreatedAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 2. Alter a table
ALTER TABLE Users ADD COLUMN Phone VARCHAR(15);

-- 3. Delete a table
DROP TABLE Users;

3 DML: Querying Data (SELECT)

The SELECT statement is the most used part of SQL.

sql 
-- Basic Selection
SELECT Username, Email FROM Users;

-- Filtering with WHERE
SELECT * FROM Products 
WHERE Price > 50 AND Status = 'Active'
ORDER BY CreatedAt DESC
LIMIT 10;

üí° Pro-Tip: SELECT *

In production, avoid using SELECT *. Explicitly naming columns reduces load and prevents breaking changes when the schema evolves.

4 DML: Modifying Data (CRUD)

CRUD stands for Create, Read, Update, and Delete.

sql 
-- INSERT: Create new records
INSERT INTO Users (Username, Email) 
VALUES ('sikandar', 'info@coderunner.pro');

-- UPDATE: Modify existing records
UPDATE Users SET Email = 'new@mail.com' 
WHERE UserID = 1;

-- DELETE: Remove records
DELETE FROM Users WHERE Status = 'Inactive';

5 Mastering JOINs

JOINs allow you to query data across related tables.

Join Type Description
INNER JOIN Returns records that have matching values in both tables.
LEFT JOIN Returns all records from the left table, and the matched records from the right table.
RIGHT JOIN Returns all records from the right table, and the matched records from the left table.
sql 
SELECT Orders.OrderID, Users.Username
FROM Orders
INNER JOIN Users ON Orders.UserID = Users.UserID;

6 Aggregations & Grouping

Use aggregate functions to perform calculations on multiple rows.

sql 
-- Count total users per city
SELECT City, COUNT(UserID) as TotalUsers
FROM Users
GROUP BY City
HAVING COUNT(UserID) > 5;
